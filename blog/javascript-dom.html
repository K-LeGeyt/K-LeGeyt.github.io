HTML, CSS, Javascript and the DOM

How do the languages of the web work together?

25 November 2021

HTML, CSS and JavaScript - languages of the web.
Just how do HTML and CSS work together to make a document we can view on a web browser or through an app. If we think of the human body and which has the basic structure of a skeleton with muscles and organs attached, this is what the Hyper Text Markup language or HTML is of a web document. It provides all the raw data or bones of the information to be displayed.

CSS (Cascading Style Sheets) then controls what that data looks like, if we go back to the human body analogy, the CSS would control the hair & eye colour, the clothes being worn and any make-up enhancing the features.

If we want to go one step further and bring JavaScript into this analogy we could think of it as the brain controlling the central nervous system which tells us how to move and interact with the world around us.

The DOM - connecting everything together.
The Document Object Model (DOM) is a programming interface which is language neutral and allows the CSS & JavaScript languages to access and interact with the HTML language. Put more simply it is the glue that holds them all together. The DOM is a bridge between the documents allowing the JavaScript to manipulate the elements in the HTML but also sending information back to the JavaScript document when the user interacts with elements so it can change the elements in response. A two way bridge all happening at lightening fast speed where things happen on either side in response to what is coming over the bridge and going back.

To be able to view a webpage on the internet we first need to access it through a browser window and it is the browser that employs the DOM to provide the JavaScript with the means to do this. You can also access the DOM of a web document using DevTools in you browser. This allows web developers to manipulate the html with CSS and JavaScript within the browser. Web developers can use this to help with de-bugging any issues with style or interactivity they are having and getting an immediate visual feedback of the change they have made.

JavaScript fundamentals.

Control flow and loops
The control flow is the order with which a computer or device runs through a script which is from the first line to the last line or more simply start to finish. This flow can be interrupted by loops inside the script which require a process to be repeated until a condition is satisfied. If we look at the process of making a cup of tea the flow of control would be get cup, add tea bag then hot water however if you prefer your tea with milk and sugar you would need to add a loop after adding these items where you stir. You might just stir 4 times in order to mix in the milk or you might need to stir until the sugar is dissolved. These repetitive actions work similarly to loops in a script.

The difference between accessing data from arrays and objects.
Objects and arrays are both ways to store collections of data in Javascript. Objects are used to represent a thing that has a set of characteristics. They can be used to store details about a person, a book, a movie or anything else that can be described with a category and a value. Arrays are a way to store a list of multiple items. They are most useful for ordered collections where items need to access by a numerical position on a list like songs on a playlist.

Objects store properties that consist of a key and a value. To access the value of the property you need to use the key.

Below we have the object variable called bookData.

                  
const bookData = {
  title: 'Harry Potter and the Prisoner of Azkaban',
  author: 'J K Rowling',
  isbn: 9780545582933,
  binding: 'Paperback'
}

To access the title of the book we need to call the bookDataObject and the key.

                  
console.log(bookData.title)

This displays: Harry Potter and the Prisoner of Azkaban

Since arrays are list-like objects you can only access the values stored in them by their place in the list. Arrays use a zero-based indexing which means the first item in the array has an index of 0.

Below we have an array which lists first, second and third.

                  
let placeValue = ['first', 'second', 'third']

To access the first value we call it by the index number which starts at zero.

                      
console.log(placeValue[0])

This displays: first

Arrays can be made up of key-value pairs, nested arrays and a mix of datatypes but it still all comes back to the zero based indexing system in order to access them.

Functions - what are they and why are they so useful?
Functions are blocks of code designed to perform a specific task when they are called to do so. They can be reused so you only have to create the function once and then you can use it multiple times putting in different arguments to get different results. They also increase the readability of code as you can decipher what the function is doing.

Below is a function that has been written to greet someone by name.

                  
function greet(name) {
    console.log("Hello " + name + "!")
}

If we assign a value to the 'name' variable and call the function it will say 'Hello' plus the 'name' and put an exclamation mark on the end.

                  
let name = prompt("Enter a name: ")
greet(name)

If I enter the name: Krissy
This will display: Hello Krissy!
