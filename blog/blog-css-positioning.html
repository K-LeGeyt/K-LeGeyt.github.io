CSS positioning - what's the difference?

Relative, absolute and fixed positioning.

15 November 2021

When setting the position of an html element, class or id in CSS you are telling the browser how to place that item visually on the web page. There are six values that the position property can have: static (default), relative, absolute, fixed, sticky and inherit. The static value is the default position for an element and will keep it in the normal flow of the page, any further settings of position added in CSS will have no effect. Often when creating a web page you want to place an element in a specific position or have it move within a responsive web page in a relative position, this is when the position value needs to be set.

It is important to note that setting position usually relates to the parent container of the element you are positioning. Only in the case of fixed positioning is the position related to viewport or the browser window. This means you can set the position of a parent element (outer container) as relative and then set the position of the child element (inside the container) to absolute based on the relative position of the parent element. You could think of these element containers a bit like Russian nesting dolls where you have dolls of decreasing size placed one inside another. You could also have several dolls next to each other placed inside a toy box. In this example the toy box then becomes the parent container. The parent container needs to have a set position other than static for the absolute value to not use the page as the container. Confused yet? I am so let's see if we can unpack this...

First up let's look at the relative position. The term 'relative' actually relates to the elements original position in the flow of the web page or parent element and lets you move it from this by setting values. You would use this setting to move an element from where it would normally appear in the flow but its position is still affected by the elements surrounding it like a static element. This position also allows you to set a z-index value which means you can move it in front or or behind other elements.

position: relative;

Below is an element with position value set to relative; is has been moved 50px to the left of its realtive position which would have been inline with the left edge of this text:

This div element has position: relative;

To set this position the following code is used in CSS*:

              
div .relative {
  position: relative;
  left: 50px;
  }

*Please note the other styling of the div elements has been ommited from this code.

Absolute position means you remove the element from the flow and set its position based on the parent element or web page and it will not affect other elements surrounding it and they won't affect its positioning. Basically the element has a set position and all the other elements will ignore it. You can also set the z-index of this position.

position: absolute;

Below is an element with position value set to relative; is has been moved 50px to the left of its realtive position which would have been inline with the left edge of this text:

This div element has position: relative;
This div element has position: absolute;

To set this position the following code is used in CSS*:

              
div .relative {
  position: relative;
  left: 50px;
  }

div .absolute {
  position: absolute;
  bottom: 0;
  right: 0;
  }

*Please note the other styling of the div elements has been ommited from this code.

A fixed position element is fixed to the web browser window or viewport. This means that as you scroll through the web page this element stays locked in place. Often used for a navigation bar you want to stay visible or a footer you want at the bottom of the page. The tricky thing with fixed position elements is that they can overlap other elements and you have to allow space to prevent this from happening if you don't like the layout.

position: fixed;

In the bottom corner of this webpage is a fixed element. You may have noticed it as you have been scrolling through this post.

This div element has position: fixed;

To set this position the following code is used in CSS*:

              
div .fixed {
  position: fixed;
  bottom: 0;
  right: 0;
  }

*Please note the other styling of the div elements has been ommited from this code.

Sticky position is a combination of both a static position and a fixed position. Often the element will have a parent element with another element above it and as you scroll down the previous element disappears and the sticky element then gets stuck in a fixed position. You often see it used for navigation which has a header above. It is important to note that this is an experimental value which means that is is not supported by all the major browsers.

The position value doesn't cascade (not inherited from the parent element) so the inherit position value forces the value to be inherited from the parent.
